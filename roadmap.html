承器 · 延伸與擴展路線圖

Extension & Expansion Work Items · 2026 → 2034

承器品牌色彩系統已完成定版，並以下一程的長弧為視角，展開七個階段、三十五個工作項目的延伸計畫。本路線圖不是全部必做，而是可選擇性展開的地圖——P0 為不可省略的基礎、P3 為選擇性擴展；S/M/L/XL 為工作量估計。階段可並行、可跳躍，依資源與時機調度。

PHASE 01 基礎完成度 0-30 days

Foundation Completion — 把品牌系統的地基收平，所有後續擴展都依賴這五項產出。

1.1
Color Token 完整規格書 COLOR TOKEN SPEC
HCT tonal scale × 13 階 × 7 色 = 91 個精確色值，涵蓋 light/dark mode、hover/pressed/disabled 狀態、focus ring。輸出 JSON + CSS Variables + Tailwind config + Figma variables 五格式。
P0M
1.2
Typography Pairing 系統 TYPE PAIRING
Noto Serif TC / Inter / JetBrains Mono 的使用規則、type scale（H1–H6 + body + mono + caption）、CJK vs Latin 字距、Outlook-safe 信件排版規則。
P0S
1.3
Brand Voice × Color Map VOICE × COLOR
每個色彩對應的語氣、場景、用詞、禁區。四種聲音模式（個人 / 顧問 / 媒體 / 前瞻）的具體語彙庫，TC/EN 雙語對照。
P0M
1.4
反模式手冊 ANTI-PATTERN GUIDE
20-30 個「不該這樣做」的具體情境——色彩誤用、字體誤用、排版誤用、聲音誤用。也作為 AI 協作時的對齊指南。
P0S
1.5
設計變數輸出 DESIGN VARIABLES
tailwind.config.js、:root{} CSS、Figma variables JSON——可直接 drop-in 任何 Next.js / React / Figma 專案。
P0S
PHASE 02 身份延伸 30-90 days

Identity Extension — 從色彩延伸到完整視覺身份，讓品牌有形狀而非只有調性。

2.1
Logo / Wordmark 系統 LOGO SYSTEM
承器主視覺標記（字符 + 幾何）、NarraTect Advisory 企業標記、Rockie 個人署名標記，以及三者之間的使用規則與共存邏輯。
P1M
2.2
自訂 SVG 符號詞彙 CUSTOM SVG VOCABULARY
因應「不使用 icon library」原則，建立 40-60 個可重複使用的幾何符號。分類：方向、狀態、類別、動作。所有未來交付物共用。
P1L
2.3
攝影方向指南 PHOTO DIRECTION
個人攝影作為品牌資產的使用規則——光線、構圖、色溫、後製風格。人像、場景、物件三種子類。
P1M
2.4
排灣幾何圖樣庫 PAIWAN PATTERN LIBRARY
從現有視覺 token 擴展為完整 pattern library，可作為書封、信封、包裝、橫幅裝飾。包含文化敏感度自審流程。
P2M
2.5
印刷品系統 PRINT COLLATERAL
信頭、信封、發票模板；名片系統（個人 × NarraTect × 雙面）；包裝、貼紙、印章規格與輸出檔。
P1M
PHASE 03 對外應用 90-180 days

Outward Applications — 所有外部可見的品牌觸點，讓市場能辨認、記得、回訪。

3.1
NarraTect Advisory 官方網站 NARRATECT SITE
Next.js + 承器設計系統。方法論展示、顧問服務、案例、聯絡。雙語 TC/EN、可申請合作的前台流程。
P0XL
3.2
個人作品集進化 PERSONAL PORTFOLIO
從現有 dark-canvas profile 升級為承器版本——攝影作品 × 顧問交付物 × 思想文章三層結構。
P1L
3.3
電子報模板升級 NEWSLETTER UPGRADE
現有新聞報刊系統套入承器色彩。Outlook-safe MJML 版本、六語言並行、RSS + email 雙發送。
P1M
3.4
LinkedIn / 社群視覺系統 SOCIAL VISUAL KIT
Header banner、貼文模板、Article 封面。保持各平台演算法的最佳寬高比，與承器一致的視覺語彙。
P1M
3.5
Keynote / Deck 主模板 KEYNOTE MASTER
涵蓋 10 種內建情境（開場、論點、案例、數據、CTA、致謝...）的 master slide。中英雙語版本、PPTX + Keynote 雙格式。
P0M
3.6
案例研究模板 CASE STUDY TEMPLATE
顧問案例的標準呈現格式。前置條件 / 介入 / 結果 / 學習四段結構，可匿名化版本。
P1M
PHASE 04 體驗與觸點 180+ days

Experience & Touchpoints — 客戶旅程中的每個微時刻，從啟動到告別都有承器的手感。

4.1
客戶啟動包 ONBOARDING PACKET
首次合作的歡迎文件。包含方法論簡介、合作節奏、溝通規範、首次提案模板。
P2M
4.2
工作坊 / 訓練教材系統 WORKSHOP KIT
講義封面、白板模板、練習表、回饋表。適用 LUMA / HCD / 變革採用三種場景。
P2L
4.3
提案書系統 PROPOSAL SYSTEM
顧問提案的標準格式。Executive summary、範疇、交付物、報價、時程的五段式結構，支持 PDF / 線上兩種形式。
P1M
4.4
信件與簽名系統 CORRESPONDENCE
Email signature（正式版 / 快速版）。常用信件模板——寒暄、跟進、拒絕、推薦四大類。
P2S
4.5
禮品 / 儀式包 GIFT & CEREMONY
客戶週年、合作啟動、感謝禮。結合家業敘事、工藝質感、承器色系。每年固定兩檔設計。
P3M
PHASE 05 思想資產化 Long-term

IP Assetization — 把你已經在做的，變成可以被引用、被授權、被傳承的資產。

5.1
承器方法論白皮書 METHODOLOGY PAPER
把這整套色彩哲學擴寫為可引用的編輯級出版品。TC/EN 雙語、PDF 與印刷雙輸出，長期可考慮 ISBN。
P0L
5.2
IP / 商標保護 TRADEMARK
「承器」「NarraTect」「個人署名」的商標申請。台灣 + 美國 + 歐盟的基本覆蓋，優先順序依據合作市場。
P1M
5.3
思想領導出版平台 EDITORIAL PLATFORM
NarraTect 官網下的 editorial 層。月刊或季刊節奏、TC/EN 雙版、可訂閱、RSS + email。
P1L
5.4
雙語出版計畫 BILINGUAL PUBLICATION
長期寫作計畫轉為書籍 / 專欄。主題方向——跨傳統求道、人本設計、組織變革敘事。
P2XL
5.5
板級定位資料集 BOARD POSITIONING
為未來進入董事會所需的背景資料庫——經歷、著作、演講、顧問案例、推薦人系統。
P1M
PHASE 06 系統整合 Long-term

Ecosystem Integration — 讓 AI 與工具鏈都認得你的品牌，不是事後套皮而是系統級對齊。

6.1
Notion 工作區視覺升級 NOTION RESKIN
把現有知識系統、資料庫、專案全部套入承器。模板升級、emoji 替換為自訂 SVG、icon 統一。
P2M
6.2
n8n 工作流視覺系統 N8N VISUAL
Workflow 通知、報告、摘要的品牌化輸出。即使是內部自動化，輸出品質也保持編輯級。
P3S
6.3
AI Agent 視覺人格 AI PERSONA UI
未來顧問 AI agent 的 UI persona。對話介面、回覆風格、視覺語彙。為客戶端自動化體驗預留設計容器。
P2M
6.4
Slack / Teams 身分系統 MESSAGING IDENTITY
Emoji 集、狀態訊息模板、署名規範。跨組織協作時的一致性。
P3S
6.5
GitHub / 開發工具品牌化 DEV TOOL BRANDING
Profile README、Repo banner、Organization 設定。對開發者社群與技術合作夥伴的可見度。
P3S
PHASE 07 文化與傳承 Selective · Long-term

Heritage & Cultural Transmission — 品牌之外的延伸，是否展開完全選擇性，不展開也不影響核心系統。

7.1
家業視覺系統 FAMILY BUSINESS SYSTEM
若家業承接發生，為其建立承器分支品牌。包裝、店面、線上通路的視覺語彙。共用承器基因但有獨立身份。
P3L
7.2
服務 / 公益計畫識別 SERVICE INITIATIVE
後續公益或文化計畫的視覺延伸。共用承器但放寬色彩比例，讓對象有主體性。
P3M
7.3
跨傳統哲學寫作平台 PHILOSOPHY PLATFORM
長期寫作計畫——跨傳統哲學觀察。可能演化為獨立子品牌，視內容深度與發表頻率而定。
P3L
7.4
攝影 / 旅行紀錄專案 PHOTO / TRAVEL PROJECT
個人攝影作品的承器應用——旅行紀錄、展覽合作、出版合作的視覺基礎。
P3M
7.5
雙語出版實驗 BILINGUAL EXPERIMENTS
短篇、隨筆、觀察的定期發表。以承器為視覺容器的長期發表計畫，可能滋養 Phase 5 的正式出版。
P3L
使用說明 · 這份路線圖是活文件，不是合約。優先順序可隨著市場信號、個人能量、合作機會調整。P0 項目是品牌系統真正成熟所需的最小可行集合，建議優先。P3 項目是生涯光譜的可能延伸，選擇性展開或完全不展開都合理。

閱讀建議 · 每季初重讀一次，問三個問題——(1) 上一季完成了什麼？(2) 下一季想動哪一項？(3) 有沒有新的項目該加進來？這份文件應該每年至少更新一次版本。

密碼管理 · 此頁面使用客戶端密碼保護（sessionStorage）。適合個人 / 小圈子使用，不適合存放敏感商業機密。若未來需要更強保護，可升級為 Vercel Password Protection 或 Cloudflare Access（需 Pro/Enterprise 方案）。
