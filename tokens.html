Phase B · Tokens in Context

承器 Tokens, 活在真實介面裡

四個情境 · 八組對比 · 從色票走到應用

色票脫離應用情境時，很難判斷對錯。這一頁用四個真實情境——editorial 長文、case study 卡片、提案封面、eDM——各自做一組 A/B 對比，讓每一個 semantic token 的選擇都有可檢視的張力。每組對比的兩個版本都成立，差在氣質取捨。

閱讀方式

  1. 每個情境都有「對比的核心選擇」——那是真正要你決策的地方，其他細節都是陪襯。
  2. A / B 兩個樣張內容相同、semantic token 配置不同。先看整體感覺，不要放大看細節。
  3. 樣張下方的 Token Mapping 表格，列出這個版本用了哪些 semantic token。
  4. 回饋時請指「scene N, version A 的 xxx」——我能精準對到要調的 token。
Scene 01 Editorial Article

思想領導長文

內文文字該用哪種暖度？
高明度醒目 vs 中明度雜誌級

這是承器最核心的使用場景——NarraTect 思想領導、個人評論、顧問交付物的敘事段落。對比的核心：body text 用 linen 主體（高對比、眼睛快速抓取）還是 stone 漂移（中對比、雜誌級氣質、讀者慢下來）。不是哪個好，是想召喚什麼閱讀節奏。

Version A
亞麻白主體

高對比閱讀體感。適合 LinkedIn 文章、email 長文、想被快速掃讀的內容。

十年前我剛進入永續領域時，合規文件是一疊厚厚的 PDF，裡頭塞滿 boilerplate。十年後仍是同一疊 PDF，只是換成更多縮寫。但如果只看合規，我們看不到組織為什麼要這樣做、為了誰、留下什麼。

合規不該是框架的堆疊，而是組織選擇的痕跡——留給下一個十年的線索。

IRIS+ 的指標不是為了填表，是為了讓投資人能問更好的問題。IMP 的五維度不是方法論，是讓影響力可被辯論。當我們把這些框架降格為 spreadsheet，我們其實是在拒絕組織的真實性。

Scene 02 Case Study Card

顧問案例數據

數據強調用哪一色？
Brass 長期承諾 vs Madder 當下張力

顧問案例的核心數字——ROI、時程、採用率——應該承載什麼情緒？銅璽代表長期價值、穩定趨勢、已兌現的承諾；茜紅代表需要此刻決策的張力、轉折點的迫切、conviction。同樣的數字，染兩種色調，讀起來是兩種案例。

Version A
Brass 長期承諾

銅璽染色適合已實現的成果、長期 ROI、續約案例。氣質：穩定、已然、值得信賴。

Scene 04 eDM

電子報

底色選哪個？
墨玄品牌一致 vs 亞麻白 Outlook 相容

eDM 是所有情境裡最棘手的——A 版堅持夜色，品牌氣質完整但 Outlook 會把深色覆蓋成白底（部分客戶端）；B 版改走亞麻白底，失去夜色氣質但跨客戶端一致性最高。真實選擇往往不是 A 或 B，而是什麼場合走哪一套——這組對比會幫你看到取捨本身。

Version A
Dark canvas

夜色底，與主站品牌完全一致。最適合：自有訂閱清單、讀者都是 Gmail/Apple Mail 用戶、高端顧問客戶訂閱制。

回饋方式

看完四個情境八組樣張後，建議你用以下方式告訴我要調整什麼：

  • 整體感：某個情境的 A 或 B 更接近承器，另一個不對勁（請說哪裡不對勁）
  • 特定元素：例如「scene 02A 的數字太暗，需要更亮」、「scene 03B 的 kicker 間距太密」
  • 缺失情境：還有什麼真實使用場景是這四個沒涵蓋但你會用到的
  • Token 層決策：看完覺得 semantic token 的分類不夠用、或有冗餘
